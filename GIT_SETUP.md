3. **Create Repository**
   - Click "Create repository"

4. **Connect and Push**


## Git Workflow

### Good Commit Messages

Follow this format:
```
Type: Brief description

Longer explanation if needed
```

**Types:**
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `style:` Formatting, missing semicolons
- `refactor:` Code restructuring
- `test:` Adding tests
- `chore:` Maintenance

**Examples:**
```bash
git commit -m "feat: Add double-click to auto-move to foundation"
git commit -m "fix: Prevent dragging face-down cards"
git commit -m "docs: Update deployment guide with shuffle feature"
```

### Branching for Features

For bigger changes, use branches:

```bash
# Create feature branch
git checkout -b feature/poker-game

# Work on poker game...
git add .
git commit -m "feat: Add poker game scaffolding"

# Push branch
git push -u origin feature/poker-game

# Merge when done (on GitHub via Pull Request)
# Or locally:
git checkout main
git merge feature/poker-game
git push
```

## Ignoring Files

The `.gitignore` is already set up to exclude:

**Unity:**
- Library/ (Unity cache - very large!)
- Temp/ (temporary files)
- Builds/ (WebGL builds - rebuild as needed)
- obj/ (build artifacts)
- .csproj, .sln files (IDE-specific)

**React Native:**
- node_modules/ (npm packages - reinstall with `npm install`)
- .expo/ (Expo cache)
- dist/ (build output)
- Build files in public/ (generated by deploy.sh)

**General:**
- .DS_Store (Mac system files)
- *.log (log files)
- .env (secrets)

This keeps your repo size small (~20-30 MB instead of several GB).

## Troubleshooting

### Issue: "Repository not found"

**Solution:**
- Check URL: `git remote -v`
- Update if wrong: `git remote set-url origin NEW_URL`

### Issue: "Permission denied"

**Solution:**
- Use HTTPS instead of SSH
- Or set up SSH keys: https://docs.github.com/en/authentication

### Issue: "Repository too large"

**Solution:**
- Check .gitignore includes Library/ and node_modules/
- Remove if accidentally added:
  ```bash
  git rm -r --cached unity-games/CardGames/Library
  git commit -m "Remove Unity Library folder"
  git push
  ```

### Issue: "Merge conflict"

**Solution:**
```bash
# Pull latest changes
git pull origin main

# Fix conflicts in files (VS Code highlights them)
# After fixing:
git add .
git commit -m "Resolve merge conflicts"
git push
```

## Quick Command Reference

```bash
# Status
git status              # See changes
git log --oneline       # See commits
git remote -v           # See remotes

# Making changes
git add .               # Stage all changes
git commit -m "msg"     # Commit
git push                # Push to GitHub

# Undoing
git restore FILE        # Discard changes
git reset HEAD~1        # Undo last commit (keep changes)
git revert HEAD         # Undo last commit (create new commit)

# Branches
git branch              # List branches
git checkout -b NAME    # Create branch
git merge NAME          # Merge branch

# Syncing
git pull                # Get latest from GitHub
git fetch               # Check for updates
git clone URL           # Copy repo
```
